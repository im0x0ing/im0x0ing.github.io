操作系统期末

Jan 03, 20267 min read

计算大题

  1. 进程调度算法：
    ◦ 重点：熟练绘制 FCFS、SJF（抢占与非抢占）、优先级调度（抢占）、RR（时间片轮转） 的甘特图。
    ◦ 要求：准确计算平均等待时间和平均周转时间。
  2. 信号量同步（PV操作）：
    ◦ 重点：掌握 生产者-消费者、读者-写者、哲学家进餐、医生看病、会议室管理 等经典模型。
    ◦ 注意：理解信号量初值设定及 PV 顺序（特别是同步与互斥相邻时，先执行同步 wait）。
  3. 内存管理与分页架构：
    ◦ 重点：二级页表（甚至三级）的地址转换，计算页号（Page Number）、页内偏移（Offset）。
    ◦ 关键计算：有效访问时间（EAT），需考虑 TLB 命中率、内存访问时间及缺页处理时间。
  4. 死锁避免（银行家算法）：
    ◦ 重点：根据 Max、Allocation 矩阵计算 Need 矩阵，判断系统是否处于 安全状态，并给出一个安全序列。

存储与文件

文件系统和磁盘调度在选择题和中型计算题中出现频率极高。

  1. 页面置换算法：
    ◦ 重点：FIFO、LRU、OPT（最佳置换） 的置换过程及缺页数计算。
    ◦ 考点：理解 Belady 异常（FIFO 特有）及 抖动（Thrashing） 现象。
  2. 文件分配方式与索引：
    ◦ 重点：计算 混合索引（如 Linux inode 的 12直接+一级+二级+三级索引） 能管理的最大文件大小。
    ◦ 对比：连续分配（Contiguous）、链接分配（Linked）与索引分配（Indexed）的优缺点（如：连续分配不支持动态增长，链接分配不支持随机访问）。
  3. 磁盘调度算法：
    ◦ 重点：SSTF（最短寻道时间优先）、SCAN（电梯算法）、C-SCAN 的移动距离计算。
    ◦ 特性：SSTF 可能导致饥饿，而 C-SCAN 提供更均匀的等待时间。

进程线程 linux

第三优先级：进程、线程与 Linux 特色（选择与填空）
此部分内容多见于小测和回忆卷的选择题，侧重概念理解。

  1. 进程与线程模型：
    ◦ 考点：线程共享资源（堆、全局变量、代码段），不共享（栈、寄存器、Thread ID）。
    ◦ 多线程模型：many-to-one（一个阻塞全部阻塞）、one-to-one、many-to-many。
    ◦ 系统调用：fork() 创建进程（注意返回值，父进程返回子 PID，子进程返回 0）、exec() 替换进程映像。
  2. 操作系统结构与机制：
    ◦ 考点：双模式执行（用户态 vs 内核态），特权指令的定义，触发状态切换的事件（异常、中断、陷入/系统调用）。
    ◦ 中断处理：中断向量、DMA（不经过 CPU 直接交换数据）、软中断（Trap）。
  3. Linux/Windows 特色：
    ◦ Linux：task_struct 结构、CFS 调度器、/proc 文件系统（伪文件系统）、内核模块加载命令（insmod, lsmod） [26, 20.7, 2951, 3195]。
    ◦ RAID 级别：RAID 0（条带化）、RAID 1（镜像）、RAID 5（分布式奇偶校验，兼顾容量与容错）、RAID 1+0。

低频

  1. 操作系统目标：方便性、效率。
  2. 存储体系：寄存器 > 缓存 > 主存 > 辅存的层次与速度对比。
  3. 微内核与宏内核对比：微内核更易扩展、更安全，但通信开销大。
  4. 地址绑定时机：编译时、加载时、执行时（动态重定位）。

面试导向

Go 的杀手锏是高并发（Goroutine），而理解 Goroutine 的前提是理解操作系统原本的进程/线程模型。

1. 进程、线程与协程 (Process, Thread, Coroutine)

  • 基础概念： 进程和线程的区别（资源分配 vs 调度单位）。

  • 上下文切换 (Context Switch)： 进程切换 vs 线程切换的开销区别。

    • Go 关联： 为什么 Goroutine 切换成本极低（用户态切换，只需保存极少寄存器）？

  • 进程间通信 (IPC)： 管道、消息队列、共享内存、信号量。

    • Go 关联： Go 推荐 “不要通过共享内存来通信，而要通过通信来共享内存” (Channel)，这对应了哪种 IPC 思想？

2. I/O 模型 (必考核心)

这是 Go 后端面试中最硬核的部分，因为 Go 的高性能主要源于此。

  • 五种 I/O 模型： 阻塞 I/O、非阻塞 I/O、I/O 多路复用、信号驱动 I/O、异步 I/O。
  • IO 多路复用 (IO Multiplexing)：
    • select, poll, epoll 的区别： 为什么 epoll 在高并发下性能更好？（红黑树、就绪链表）。
    • 触发模式： 水平触发 (LT) vs 边缘触发 (ET)。
    • Go 关联： Go 的 Netpoller（网络轮询器）底层就是基于 epoll (Linux) / kqueue (Mac) / IOCP (Windows) 实现的。面试常问：Go 如何用同步的代码逻辑实现了异步的 I/O 性能？

3. 内存管理

  • 虚拟内存： 什么是虚拟内存？分页 (Paging) 和分段。
  • 用户态 vs 内核态： 系统调用 (System Call) 会导致状态切换。
  • 堆 (Heap) vs 栈 (Stack)：
    • Go 关联： 逃逸分析 (Escape Analysis)。在 C/C++ 中你需要手动管理堆内存，但在 Go 中，编译器决定变量是在栈上还是堆上。你需要复习一下什么情况会导致变量“逃逸”到堆上（如返回局部变量指针）。

4. 死锁 (Deadlock)

  • 产生死锁的四个必要条件： 互斥、占有且等待、不可抢占、循环等待。
  • 如何预防/避免： 破坏其中一个条件。
    • Go 关联： RWMutex, Channel 如果使用不当会导致死锁（fatal error: all goroutines are asleep - deadlock!）。
